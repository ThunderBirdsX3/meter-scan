# PRD-fuel-log — Fuel Log (บันทึกการเติมน้ำมัน)

## 1. Vision

แอปบันทึกการเติมน้ำมันบนมือถือ ที่เปิดมาใช้ได้ทันที ไม่ต้องสมัครบัญชี ข้อมูลทุกอย่างอยู่ในเครื่อง (SQLite) ผู้ใช้บันทึกการเติมน้ำมันได้ทั้งกรอกเองและสแกนมิเตอร์ตู้จ่าย แล้วเห็นภาพรวมค่าใช้จ่าย/ปริมาณ/ประสิทธิภาพรถได้ราย trip ราย เดือน และรายคัน

## 2. Problem

คนใช้รถจดค่าน้ำมันในกระดาษ/แชต/แอปโน้ต กระจัดกระจาย คำนวณยอดรวมและ กม./ลิตร เองลำบาก แอปบันทึกค่าน้ำมันส่วนใหญ่บังคับสมัครบัญชีหรือ sync ขึ้น cloud ทำให้เปิดใช้ครั้งแรกช้าและกังวลเรื่องความเป็นส่วนตัว การกรอกตัวเลขมิเตอร์จากตู้จ่ายด้วยมือก็ผิดง่าย

## 3. Personas

- **เจ้าของรถส่วนตัว** — ขับรถประจำวัน อยากรู้ค่าน้ำมันรายเดือนและ กม./ลิตร โดยไม่ยุ่งยาก
- **คนมีหลายคัน** — รถบ้าน + มอเตอร์ไซค์ อยากแยกค่าใช้จ่ายรายคัน
- **คนเดินทางเป็นทริป** — อยากสรุปค่าน้ำมันต่อทริป (เที่ยวต่างจังหวัด) ที่อาจเติมหลายครั้ง

## 4. Goals

- บันทึกการเติมน้ำมัน 1 ครั้งได้ภายในไม่กี่วินาที (กรอกเอง หรือสแกนช่วยกรอก)
- เปิดแอปใช้ได้เลย ไม่มีบัญชี ไม่ต้องต่อเน็ต ข้อมูลไม่ออกจากเครื่อง
- จัดการรถของตัวเอง (เพิ่ม/แก้/ลบ) และจัดกลุ่มการเติมเป็นทริปได้
- เห็นภาพรวมค่าใช้จ่าย/ปริมาณ/จำนวนครั้ง/ประสิทธิภาพ ราย trip · เดือน · รถ
- ลดการกรอกผิดด้วยการสแกนเลขมิเตอร์ LCD จากรูปถ่าย

## 5. Non-goals

- บัญชีผู้ใช้ / multi-user / แชร์ข้อมูลข้ามเครื่อง
- รองรับมิเตอร์แบบ mechanical rolling-digit (สแกนรองรับ 7-seg LCD เท่านั้น — ดู [[SRS-meter-scan]])

## 6. Success Metrics (SC-###)

- **SC-001** — บันทึกการเติม 1 ครั้ง (กรอกเอง) เสร็จได้ ≤ 30 วินาที วัดด้วย usability run (เวลาเฉลี่ย 5 คน)
- **SC-002** — หน้า overview โหลดและคำนวณเสร็จ ≤ 1 วินาที ที่ข้อมูล 1,000 fuel entries วัดด้วย in-app timing
- **SC-003** — ข้อมูลไม่หายข้ามการปิด/เปิดแอป (0% loss) วัดด้วย restart test หลังบันทึก N รายการ
- **SC-004** — ใช้งานครบทุก flow แบบ offline 100% (เปิด airplane mode) ไม่มี network call วัดด้วย network inspector
- **SC-005** — สแกนมิเตอร์อ่านเลข Amount/Liters/Price ถูกต้องครบทุกหลัก (exact) บน eval set วัด field-accuracy (ดู [[SRS-meter-scan]])

## 7. User Stories Overview (Prioritized)

### US1 — บันทึกการเติมน้ำมัน (กรอกเอง) (Priority: P1) — MVP

**As a** เจ้าของรถ **I want** บันทึกการเติม (วันที่ ลิตร ราคา/ลิตร ยอดเงิน แบรนด์ ประเภทน้ำมัน เลขไมล์) **so that** มีประวัติค่าน้ำมันครบ

**Why this priority:** หัวใจของแอป — ถ้าบันทึกไม่ได้ feature อื่นไม่มีความหมาย

**Independent Test:** เปิดแอปครั้งแรก → เพิ่ม fuel entry กรอกมือ → ปิด/เปิดแอป → entry ยังอยู่

---

### US2 — ดูภาพรวม (Priority: P1) — MVP

**As a** ผู้ใช้ **I want** ดูยอดรวมเงิน/ลิตร/จำนวนครั้ง/กม.ต่อลิตร ราย trip · เดือน · รถ **so that** รู้ว่าจ่ายค่าน้ำมันไปเท่าไหร่และรถกินน้ำมันแค่ไหน

**Why this priority:** เป็นคุณค่าหลักที่ทำให้คนบันทึกต่อเนื่อง

**Independent Test:** มี entries หลายรายการต่างเดือน/ต่างรถ → เปิด overview → ตัวเลขรวมตรงกับผลรวมที่คำนวณมือ

---

### US3 — จัดการรถ (Priority: P2)

**As a** คนมีหลายคัน **I want** เพิ่ม/แก้/ลบรถ และผูกการเติมกับรถ **so that** แยกค่าใช้จ่ายรายคันได้

**Why this priority:** จำเป็นสำหรับ "รายรถ" แต่ใช้แอปได้แม้ไม่เพิ่มรถ (optional)

**Independent Test:** เพิ่มรถ 2 คัน → บันทึกการเติมผูกคนละคัน → overview รายรถแยกถูกต้อง

---

### US4 — จัดกลุ่มทริป (Priority: P2)

**As a** คนเดินทาง **I want** สร้างทริปแล้วผูกการเติมหลายครั้งเข้าทริป **so that** สรุปค่าน้ำมันต่อทริปได้

**Why this priority:** เพิ่มมุมมองสรุป แต่ optional (ใช้ได้แม้ไม่สร้างทริป)

**Independent Test:** สร้างทริป → ผูก entry 3 รายการ → overview รายทริปรวม 3 รายการถูกต้อง

---

### US5 — สแกนมิเตอร์ช่วยกรอก (Priority: P2)

**As a** ผู้ใช้ที่ขี้เกียจพิมพ์ **I want** ถ่ายรูปตู้จ่ายแล้วให้แอปเติมเลข Amount/Liters/Price ให้ **so that** กรอกเร็วและผิดน้อยลง

**Why this priority:** ต่อยอดจาก POC ที่มีอยู่ ([[FEAT-MeterScan]]) — เป็นตัวช่วย ไม่ใช่ทางเดียวในการกรอก

**Independent Test:** ถ่าย/เลือกรูปมิเตอร์ → สแกน → ค่าที่อ่านได้เด้งเข้าฟอร์ม entry → user แก้/ยืนยัน → บันทึก

## 8. Constraints

- **Deadline**: ยังไม่กำหนด <!-- TODO -->
- **Tech stack**: Ionic 8 + Angular 20 (NgModule) + Capacitor 8, iOS + Android (ดู [[REF-TechStack]]). Storage = SQLite on-device
- **Compliance**: ไม่เก็บข้อมูลบนเซิร์ฟเวอร์ — ข้อมูลทั้งหมดอยู่ในเครื่องผู้ใช้
- **Language**: UI ภาษาไทย

## 9. Assumptions

- หน่วยเงิน = บาท (THB), ปริมาณ = ลิตร, ระยะ = กิโลเมตร
- ผู้ใช้ 1 คนต่อ 1 เครื่อง (ไม่มี multi-profile)
- เลขไมล์ (odometer) ที่กรอกเป็น odometer สะสมของรถ (ใช้คำนวณ กม./ลิตร แบบ full-tank)

## 10. Risks

- **R1** — สแกนมิเตอร์อ่านผิดแล้วผู้ใช้ไม่ตรวจ → mitigation: ค่าจากสแกนเป็น draft เสมอ ต้องให้ user ยืนยันก่อนบันทึก
- **R2** — ชุดแบรนด์/ประเภทน้ำมัน built-in ล้าสมัย (แบรนด์เปลี่ยนสูตร) → mitigation: brand/type = master config read-only (Clarify 2026-06-29); อัปเดตผ่าน app version ใหม่ (append config rows). user แก้เองไม่ได้ (ไม่เปิด custom config)
- **R3** — กม./ลิตร เพี้ยนถ้า user ข้ามการเติมบางครั้งหรือเติมไม่เต็มถัง → mitigation: ระบุชัดว่าเป็นค่าประมาณแบบ full-tank; ข้าม entry ที่ odometer ว่าง

## 11. Glossary

- **Fuel entry** — 1 รายการการเติมน้ำมัน (วันที่ ลิตร ราคา/ลิตร ยอดเงิน แบรนด์ ประเภท เลขไมล์ รูป)
- **Trip** — กล่องที่จัดกลุ่ม fuel entry หลายรายการเข้าด้วยกัน (1 trip : N entries)
- **Vehicle** — รถ 1 คันของผู้ใช้
- **Brand** — แบรนด์ปั๊มน้ำมัน (เช่น PTT, Bangchak)
- **Fuel type** — ประเภทน้ำมันของแบรนด์ (เช่น แก๊สโซฮอล์ 95, ดีเซล B7)
- **Amount / Liters / Price** — ยอดเงิน(฿) / ปริมาณ(ลิตร) / ราคาต่อลิตร; **Amount = Liters × Price**
- **Meter scan** — ฟีเจอร์อ่านเลขมิเตอร์ LCD จากรูปด้วย CRNN ONNX ([[FEAT-MeterScan]])

## 13. Clarifications

### Session 2026-06-28

- **Trip model:** Trip = กล่องรวมหลายครั้งเติม (1 trip : N fuel entries; entry ผูก trip ได้แบบ optional)
- **Seed data:** ใส่ชุดแบรนด์ไทย built-in มากับแอป; user เพิ่ม/แก้เองได้
- **Report metrics:** ยอดเงินรวม(฿) + ปริมาณรวม(ลิตร) + จำนวนครั้ง + กม./ลิตร (entry มีช่องกรอกเลขไมล์)
- **Non-goals (MVP):** Export CSV/PDF, Cloud sync/backup, Reminder/แจ้งเตือน
